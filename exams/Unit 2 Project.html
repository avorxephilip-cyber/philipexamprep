Unit 2: Integration, Scope, and Time Management

Self-Assessment Questions & Answers

Session 1: Project Management Integration

1. Explain project management integration.
Project management integration involves the processes and activities needed to identify, define, combine, unify, and coordinate the various processes and project management activities within the Project Management Process Groups. It includes making choices about resource allocation and balancing competing objectives.
2. State the processes of project management integration.
  • Develop Project Charter
  • Develop Project Management Plan
  • Direct and Manage Project Work
  • Monitor and Control Project Work
  • Perform Integrated Change Control
  • Close Project or Phase

Session 2: Project Scope Management

1. Define project scope and project scope management.
Project Scope: refers to the work that must be performed to deliver a product, service, or result with the specified features and functions.
Project Scope Management: includes the processes required to ensure that the project includes all the work required, and only the work required, to complete the project successfully.
2. Explain the difference between product scope and project scope.
Product Scope: The features and functions that characterize a product, service, or result.
Project Scope: The work performed to deliver a product, service, or result with the specified features and functions.

Session 3: Work Breakdown Structure (WBS)

1. Define Work Breakdown Structure.
WBS is a hierarchical decomposition of the total scope of work to be carried out by the project team to accomplish the project objectives and create the required deliverables.
2. State any three (3) importance of WBS.
  • Structured Vision: It provides a structured vision of what has to be delivered.
  • Resource Allocation: It helps in resource allocation and responsibility assignment.
  • Better Estimation: It facilitates better estimation of costs, risks, and time.

Session 4: Project Time Management

1. Define project time management.
Project time management involves the processes required to manage the timely completion of the project.
2. State the processes of project time management.
  1. Plan Schedule Management
  2. Define Activities
  3. Sequence Activities
  4. Estimate Activity Resources
  5. Estimate Activity Durations
  6. Develop Schedule
  7. Control Schedule

Session 5: Project Activity Sequencing & Network Diagram

1. Define Activity Sequencing.
Activity sequencing is the process of identifying and documenting relationships among the project activities.

  • External dependencies

Session 6: Critical Path Method (CPM)

1. Define Critical Path.
The critical path is the sequence of activities that represents the longest path through a project, which determines the shortest possible project duration.
2. Explain Float (Slack).
Float or Slack is the amount of time that a scheduled activity can be delayed from its early start date without delaying the project finish date or violating a schedule constraint. Activities on the critical path have zero float.
